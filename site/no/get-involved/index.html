Programvare for mennesker

Bokwyrm er laget for å være liten og nør; hvis du har ønsker, feilrapoprter eller store vyer, vil vi gjerne ha tilbakemelding fra deg.

Github Feil og funksjonsønsker

BokWyrms feilliste og kode er lagret på GitHub — det er stedet å rapportere feil og legge inn funksjonsforespørsler.

Loomio Decision-making

Bigger-picture and non-technical decision-making is coordinated on our Loomio group. Everyone who uses BookWyrm is encouraged to take part.

Crowdin Oversettelser

Hjelp til med å oversette BookWyrm til andre språk! Du kan lære mer om oversettelsesprosessen i dokumentasjonen vår.

Mastodon Sosiale medier

You can follow BookWyrm on Mastodon to get updates about new features. Feel free to tag the account or send a direct message if you'd like to get in touch.

Matrix Utviklerchat

We use Matrix for BookWyrm development chat — it's a great place to ask questions and get help. It's focused on development of BookWyrm software and running BookWyrm instances, but all kinds of BookWyrm discussion is welcome.

Who maintains BookWyrm?

BookWyrm was started by Mouse Reeve - they steer the project, run the flagship instance, and manage the developer community. The project is community run and strives for a welcoming, non-heirarchical environment.

BookWyrm er delt, åpen og ikke-komersiell programvare vedlikeholdt av Mouse Reeve.
